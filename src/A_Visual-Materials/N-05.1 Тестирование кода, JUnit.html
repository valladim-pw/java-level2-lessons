* На рисунке показаны примеры применения аннотаций @BeforeClass и @AfterClass.
* @BeforeClass помогает один раз инициализировать ссылку на объект calc, чтобы не делать это в каждом методе класса.
* @AfterClass нужна для того, чтобы после использования класса, ссылка не зависла в памяти и ее мог убрать сборщик мусора (так как она не будет ссылаться ни на один объект).

Рис.34 

* Порядок быстрой генерации тестового класса используя возможности IDE:
- Кликаем по названию класса, в котором хотим что-либо протестировать,
- Кликаем по значку желтой лампочки либо Alt + Enter на клавиатуре,
- В появившемся меню кликаем Create Test,
- В появившемся окне Create Test выбираем нужные опции и метод(ы), для которых хотим создать тесты, жмем OK,
- Если тестовый класс уже существует, система спросит нужно ли его обновлять (Update existing class?), жмем OK,
- В тестовом классе генерируется тест с указанными нами параметрами.
