Рис.26 

* Хэш-функция — это алгоритм, который выполняет преобразование строки произвольной длины в число с фиксированной длиной.
* Основная задача хэш-функции состоит в том, чтобы минимизировать коллизии, то есть ситуации, когда двум различным строкам соответствует одно и то же числовое значение.
* Java предоставляет несколько встроенных хэш-функций для строк, таких как hashCode() и hash().
* Метод hashCode() — это метод класса String, который возвращает хэш-код строки (это число, которое вычисляется на основе содержимого строки).
* Метод hash() класса Objects предоставляет более эффективный способ вычисления хэш-кода строки, чем метод hashCode(), поддерживает переменное число аргументов и принимает любые объекты.
* В Java также можно определить собственную хэш-функцию для строк, реализуя интерфейс hashCode(). В примере пользовательская хэш-функция customHashFunction(String str) вычисляет хэш строки суммируя значения всех символов в строке.

Рис.28 

* Каждый примитивный тип данных в Java имеет ограниченный диапазон значений, которые он может хранить.
* Поскольку все типы целых чисел в Java (кроме char) - знаковые, то диапазон разделен на две почти равные части с отрицательными и положительными значениями (отрицательная часть больше положительной на 1). Например, тип byte имеет диапазон от -128 до 127, а тип int имеет диапазон от -2 147 483 648 до 2 147 483 647.
* Когда мы выполняем операцию, которая превышает этот диапазон, происходит переполнение.
* C помощью круговой диаграммы мы можем наглядно представить процесс переполнения для каждого примитивного типа данных. После достижения максимального значения, значение переполняется и возвращается к минимальному значению, образуя замкнутый круг.
* Чтобы предотвратить переполнение примитивных типов данных, рекомендуется:
  1. Внимательно выбирать тип данных в зависимости от ожидаемого диапазона значений. Если вы ожидаете большие числа, используйте типы данных с большим диапазоном, например, long вместо int.
   2. Проверять диапазон значений при выполнении операций и принимать соответствующие меры. Например, использовать условные операторы или проверки на допустимый диапазон перед выполнением операции.
   3. Использовать классы-обертки, такие как BigInteger или BigDecimal, которые позволяют работать с числами произвольной точности и избегать переполнения.

Рис.29 

* Портация на Java хэш-функции, написанной на C++, из статьи (Рис.27, *3).
* Поскольку в Java в отличие от C++ нет типа unsigned int (беззнаковое целое число), а все целые числа знаковые, то для правильного приведения типа был написан метод unsignedInt(long num), который возвращает тип long, преобразуя аргумент num того же типа.
* Для преобразования используется константа long UINT_MAX, значение которой закрывает полный диапазон числа типа int от минимального отрицательного до максимального положительного и равно Integer.MAX_VALUE * 2 + 1.
* Остаток от деления аргумента на UINT_MAX как раз будет беззнаковое число, не выходящее за пределы суммы диапазонов Integer.MAX_VALUE и в положительную и в отрицательную сторону.
