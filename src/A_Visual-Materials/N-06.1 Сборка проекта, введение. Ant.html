Рис.12 

* Для Windows, чтобы правильно отразилось приветствие в нашем build.xml файле нужно поменять кодировку в командной строке.
* Для этого надо перейти в каталог C:\Windows\System32 с файлом cmd.exe.
* Кодировки cmd.exe:
  - MS-DOS (по умолчанию) - 866,
  - UTF-8 - 65001,
  - Кириллица - 1251
* Узнать кодировку можно командой chcp
* Изменить кодировку можно командой chcp <нужная кодировка>, в нашем случае - chcp 1251 (рис. выше (1))
* После закрытия "Командной строки" кодировка возвращается к значению по умолчанию!
* Поэтому меняем кодировку перед запуском нашего проекта, затем переходим в каталог с нашим проектом и запускаем его командой ant (рис. выше (2)).

Рис.13 

* Порядок запуска проекта Ant из IntelliJ IDEA:
1. Кликаем Ant Build на правой стороне окна IDEA,
2. В открывшейся вкладке жмем кнопку "+",
3. В открывшемся окне Select Ant Build File выбираем наш build.xml и жмем OK,
4. Во вкладке Ant Build появляется наш проект HelloWorld и в нем наша цель (target) - hello,
кликаем на цель hello и активируется стрелка запуска проекта.
Запустить проект можно либо кликом по стрелке запуска, либо двойным кликом по цели.
5. Запустив проект, во вкладке Messages мы видим результат работы нашей сборки Ant - cообщение "Привет, Мир!" и сообщение "Ant bulild completed...".

Рис.16 

* Для Пример 1 создадим отдельный проект (рис. выше (1)).
* Основной класс проекта Main4Ant2 будет рассчитывать n-ое число Фибоначчи (рис. выше (2)).
* Файл build.xml определяет сценарий сборки проекта, который мы назовем как и основной java файл (рис. выше (3)):
- Сначала с помощью свойств определяется каталог, где будет размещен скомпилированный файл проекта Main4Ant2.class - build/classes,
- Назначение целей определяется их именами (например, назначение target name="compile" - компиляция файла, и т.п.),
- В целях возможно определить зависимости (depends), указывающие на взаимосвязь и порядок выполнения целей.
В нашем случае компиляция (compile) будет происходить после clean (удаления каталога build/classes), а выполнение (run), соответственно после компиляции,
- Внутри каждой цели размещены задания (mkdir, javac, java, delete) , которые определяют действия необходимые для выполнения целей (рис.14).
