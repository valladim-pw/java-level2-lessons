* JIT (Just-In-Time) это компиляция часто повторяющихся фрагментов кода.
* Такой код компилируется один раз и в дальнейшем для ускорения выполнения запускается в процессе интерпретации байт-кода.
* Есть 4 уровня JIT-компиляции.
* JIT-компилятор самостоятельно решает какой уровень задать в зависимости от частоты повторения кода.
* С1 и С2 - два вида JIT-компиляторов.
* C1 (клиентский) - способен компилировать до 3-его уровня сложности.
* C2 - компилирует самый сложный 4-ый уровень.

* Декомпиляция онлайн - javadecompilers.com

* В результате выполнения команды javac в заданной папке появляется бинарный файл с расширением .class.
* По команде java JVM исполняет бинарный файл.

* Создание библиотеки в Java - это создание архива.
* Ecть специальные архивы с расширением jar, в которые можно поместить байт-код.
* Если в библиотеке есть исполняемый файл (с методом main), то командой java мы можем эту библиотеку запустить
