Рис.19 

* В IntelliJ IDEA нет необходимости писать зависимости вручную. Чтобы их добавить нужно:
1. В файле pom.xml, наведя курсор на код правой кнопкой мыши вызвать контекстное меню, где нажать на Generate либо нажать сочетание клавиш Alt + Ins.
2. В открывшемся списке нажать на Dependency.
3. В открывшемся окне выбрать нужный артефакт и кликнуть Add.
4. Нужная зависимость появится в коде.

Рис.20 

* В IntelliJ IDEA также есть возможность генерировать теги, для этого надо:
1. В файле pom.xml, наведя курсор на пустое место (!), куда мы собираемся подставить тег правой кнопкой мыши вызвать контекстное меню, где нажать на Generate либо нажать сочетание клавиш Alt + Ins.
2. В открывшемся списке нажать на XML Tag.
3. В открывшемся списке кликнуть по нужному тегу.
4. В списке значений (если тег, например scope) выбрать нужное значение.

Рис.27 

* Пишем цели по созданию и запуску jar-файла.
* Название файла будет такое же как у проекта ${ant.project.name},
в атрибуте excludes="Test*.class" указываем, что исключаем из архива тестовый файл, поскольку он нужен только для тестирования кода,
в директиве manifest указываем, что необходимо создать манифест.
* В цели по запуску jar-файла пишем задание на запуск java, в котором определяем файл для запуска jar="${build}/${ant.project.name}.jar" и то, что запуск будет в отдельной копии виртуальной машины Java (fork="true").

Рис.28 

* build.xml

* Если мы запустим проект с конечным вариантом build.xml сначала будет запрошена дефолтная цель build-jar, а далее в соответствии с зависимостями указанными в целях (на рис. синие стрелки), исполнение будет происходить в обратном порядке, начиная с цели clean (зеленые стрелки).
* После этого, если нет ошибок в тесте, происходит выполнение цели run-jar, т.е. запуск jar-файла.

- На этой странице находим выпадающее меню Project Encoding и в нем выбираем windows-1251,
- Кликаем здесь же OK.
