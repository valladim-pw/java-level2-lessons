* В IntelliJ IDEA нет необходимости писать зависимости вручную. Чтобы их добавить нужно:
1. В файле pom.xml, наведя курсор на код правой кнопкой мыши вызвать контекстное меню, где нажать на Generate либо нажать сочетание клавиш Alt + Ins.

Рис.24 

* Для проекта напишем build.xml.

* build.xml

* Поскольку для IDEA есть своя система classpath-ов, мы подключили внешние библиотеки junit и hamcrest-core, но для Ant нужно расположить и указать расположение отдельно, для чего размещаем их в папке lib (рис.23) и прописываем локацию junit в properties (1), а также дописываем classpath с junit в задании javac.
* Для теста пишем цель test (2), в которой указываем зависимость от успешной компиляции depends="compile".
* В цели будет единственное задание junit, в котором указываем атрибуты haltonerror="yes" и haltonfailure="yes", которые обеспечат прерывание сценария выполнения если тест не пройдет.
Атрибут fork="true", означает, что будет запущена отдельная копия виртуальной машины Java, в которой будет исполняться тест. Если его не указать, то вылетит ошибка.
formatter обеспечивает формат вывода результатов теста, атрибут type="brief" - информация должна быть краткой, usefile="false" - результат выводится не в файл а на консоль.

Рис.27 

* Пишем цели по созданию и запуску jar-файла.
* Название файла будет такое же как у проекта ${ant.project.name},
в атрибуте excludes="Test*.class" указываем, что исключаем из архива тестовый файл, поскольку он нужен только для тестирования кода,
в директиве manifest указываем, что необходимо создать манифест.
* В цели по запуску jar-файла пишем задание на запуск java, в котором определяем файл для запуска jar="${build}/${ant.project.name}.jar" и то, что запуск будет в отдельной копии виртуальной машины Java (fork="true").
